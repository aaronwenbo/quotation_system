# 液压管接头报价管理系统 - 需求规格说明书 (PRD)

## 1. 项目概述 (Project Overview)
本系统是一套专为**液压管接头行业**定制开发的专业报价管理软件。
核心目标是解决行业内外贸销售人员在报价过程中遇到的痛点：产品规格繁杂、价格体系多变、利润核算不清以及报价单制作繁琐。
本系统同时支持两种发布形态以满足不同用户场景：
- **Web 模式/服务端部署** (Node.js + MySQL)：适用于公司团队协同使用，数据集中管控。
- **Windows 桌面免安装版** (Electron + SQLite)：提供单文件绿色版 `.exe`，无需网络，数据全部存储于本地，适合业务员独立办公或在外出差演示时使用。

## 2. 核心业务实体模型 (Data Entities)
根据系统数据库设计与业务逻辑，抽象出以下核心业务实体：
- **产品库 (Products)**：包含产品统一编码、中英文名称、规格参数 (JSON)、基础成本价、基础售价等。
- **客户档案 (Customers)**：记录客户名称、联系人、联系电话、邮箱及收货地址等基本画像。
- **报价体系/模板 (Templates)**：独立的价格配置字典，用于针对不同代理层级或单独客群设置特定单价。
- **报价单主数据 (Quotations)**：报价单号、关联客户、应用的报价体系模板、报价有效期、整单总折扣率、附加备注及单据状态。
- **报价单明细 (Quotation Items)**：报价单关联的具体产品，包含售出数量、销售单价、产品当时的成本价、行折扣及折后行总价。
- **价格历史快照 (Price History)**：作为辅助决策数据层，记录每次报价确立后产品对某客户的历史单价记录。

## 3. 功能模块需求 (Functional Requirements)

### 3.1 产品与价格库管理
- **产品基础维护**：支持产品的新增、编辑、删除、检索，支持录入产品中英文双语信息。
- **批量导入导出**：支持通过 Excel 模板大批量导入产品库；支持一键将现有产品列表导出为 Excel。
- **报价模板管理**：允许业务建立多套价格体系（如：一级代理价、海外大客户价）。每个模板支持批量导入或调整针对各类产品的特供价格。

### 3.2 客户关系辅助管理 (CRM Basis)
- **客户统一建档**：管理往来客户的信息，在开具报价单时支持快速搜索并引用对应的主体信息，减少重复打字输入。

### 3.3 报价单核心主推演流程
- **建单配置**：创建一个新的报价单时，选择目标客户并绑定适用的“首选报价模板”。
- **单据明细录入**：
  - 手工逐条检索添加产品及数量。
  - **亮点功能：Excel 询价单转换导入**：支持将客户发来的 Excel 询价明细一键导入，系统自动匹配系统产品并带入当前体系关联价格。
- **折扣规则及调整**：
  - **行级别折扣**：默认 100%（原价），业务员可针对某一行产品单独做打折（`行金额 = 数量 × 单价 × 行折扣 / 100`）。
  - **整单总折扣**：所有行金额相加得出小计后，提供全单级二次折扣控制（`最终合计 = 汇总小计 × (1 - 整单折扣率 / 100)`）。
- **成交历史比质比价**：
  - **历史倒查**：业务员可在开单时点击某个产品单价旁的快捷图标，直接弹窗/悬浮出**过去 5 次该产品成交历史价格**，辅助新单议价策略。

### 3.4 毛利动态分析反馈 (Profit Analysis)
- **精细化利润管理**：在编写报价单流程中，无论是每一行产品级别的横向数据，还是整个报价单底部的汇总层面，系统必须**实时测算并直观展示出“绝对毛利额”和“毛利率”**。（以产品表中维护的成本数据为基准）。有效防止一线业务员在多轮议价中出现接亏损订单的致命失误。

### 3.5 报表与下载 (Export & Reporting)
- **精美对外输出**：将报价单在线排版，一键导出为格式对齐、内容专业的 **PDF 报价单** 和 **Excel 电子表格** 给最终客户。
- **语言动态调整**：导出文档及对外展示单据内容支持纯中文版、亦支持纯英文版，一键切换。

### 3.6 系统平台功能
- **国际化 (i18next)**：不仅体现在打印单据，本系统本身的后台 UI 界面同样提供 中文/English 双语言一键切换，满足跨国贸易团队使用。

## 4. 非功能性与技术架构需求 (Non-Functional Requirements)

### 4.1 技术栈统一化与同构
要求同一套前端和后端逻辑能同时适配 Web（服务端）与 Desktop（客户端）场景：
- **前端**：React 19 + Vite 构建，结合 Ant Design 6 实现企业级 UI 交互。
- **后端抽象层**：基于 Node.js，数据库操作需封装适配层，使用环境变量 (`DB_TYPE`) 热切换。在服务器上使用 `mysql2` 连接真正的 MySQL，在 Electron 内嵌的 Node 环境中动态按需加载 `sqlite3` 操作本地 `.db` 文件。

### 4.2 部署运维与数据安全
- **私密安全**：单机绿色版本所有的商业敏感数据留在跑 `sqlite` 的用户本机磁盘，必须保证拔网线依然可用，并且无需业务员进行复杂的环境安装部署过程（实现真正的“点开即用”架构）。
- **打包能力**：要求提供一键打包脚本，能将前后端合体并利用 electron-packager 打包为 Windows `.exe` 单一目录产物。
