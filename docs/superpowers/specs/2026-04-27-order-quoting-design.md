# 订单报价功能设计规格

**日期：** 2026-04-27

## 1. 功能概述

对 order 文件夹中的报价文件进行自动报价，基于标准产品库匹配价格。新增0→O字符混淆容错匹配规则：客户录入错误识别并标注匹配类型。

## 2. 输入输出

### 2.1 输入文件

| 文件 | 说明 |
|------|------|
| `data/standard_product_library.xlsx` | 标准产品库（标准编码、价格、规格、原规格编码） |
| `order/20260420.xlsx` | 主订单文件（含产品型号、数量） |
| `order/HT.xlsx` | HT订单文件（仅产品编码） |

### 2.2 输出文件

| 文件 | 说明 |
|------|------|
| `output/20260420_quoted.xlsx` | 报价后的主订单文件 |
| `output/HT_quoted.xlsx` | 报价后的HT订单文件 |

## 3. 匹配规则

### 3.1 匹配流程

```
订单编码 → 清理格式（去空格） → 直接匹配标准库 → 找到则填入
        → 未找到 → O→0转换（所有字母O替换为数字0） → 再次匹配 → 找到则标注
        → 仍未找到 → 留空并标注"无匹配"
```

### 3.2 产品行识别规则

- A列非空
- 非标题行：排除 "FERRULE FOR..."、"PART NO." 等非编码行
- 编码长度通常为产品编码格式（如 00110-04）

## 4. 输出列设计

### 4.1 20260420.xlsx

| 列 | 列名 | 说明 |
|-----|--------|------|
| A | 产品型号 | 保持原内容不变 |
| F | 数量 | 保持原内容不变 |
| G | 单价 | 从标准库匹配的价格 |
| H | 总价 | 计算公式：数量 × 单价 |
| I | 标准编码 | 匹配到的标准库编码 |
| J | 原规格编码 | 标准库中的原规格编码 |
| K | 匹配标注 | "直接匹配" / "O→0转换匹配" / "无匹配" |

### 4.2 HT.xlsx

| 列 | 列名 | 说明 |
|-----|--------|------|
| A | 产品编码 | 保持原内容不变 |
| C | 单价 | 从标准库匹配的价格 |
| D | 标准编码 | 匹配到的标准库编码 |
| E | 原规格编码 | 标准库中的原规格编码 |
| F | 匹配标注 | "直接匹配" / "O→0转换匹配" / "无匹配" |

## 5. 日志设计

- 日志文件：`logs/quote_order.log`
- 日志内容：
  - 总处理行数
  - 直接匹配数量
  - O→0转换匹配数量
  - 无匹配数量
  - 无匹配编码列表（便于人工处理）

## 6. 实现方式

新建独立脚本 `src/quote_order.py`，保持与现有架构隔离，职责单一。

## 7. 统计指标

- 匹配成功率统计
- O→0转换匹配成功数量统计
