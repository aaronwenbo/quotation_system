# 产品编码统一合并工具设计

## 项目背景
初始化ERP系统，需要将两份来源不同的产品报价文件合并为一套统一完整的产品编码库，并用统一编码更新现有库存文件。

- 两份报价文件可能存在：
  - 产品编码完全一致 → 直接合并
  - 产品编码不同但实际是同一产品 → 需要人工匹配后统一
  - 格式差异（空格、分隔符等）→ 可自动清理处理

## 设计目标
1. 合并两份报价文件生成一套统一的产品编码库
2. 生成完整的原始编码→统一编码映射表，可追溯、可复用
3. 使用统一编码更新库存文件
4. 输出详细统计信息（完全一致数量、单方独有数量等）

## 处理流程

```
┌────────────────────────────┐
│  读取两份报价输入文件        │
│  - product_import_template.xlsx (用户报价)
│  - product_import_template_neixian.xlsx (内陷报价)
└────────────┬───────────────┘
             │
             ▼
┌────────────────────────────┐
│  第一步：自动格式清理统一    │
│  - 去除所有空格
│  - 统一分隔符格式
│  - 记录：原始编码 → 清理后编码 映射
└────────────┬───────────────┘
             │
             ▼
┌────────────────────────────┐
│  第二步：自动匹配合并        │
│  1. 清理后编码完全一致 → 自动合并，保留价格信息
│  2. 仅在文件A存在 → 直接保留
│  3. 仅在文件B存在 → 直接保留
│  4. 规格相似编码不同 → 标记为待人工匹配，导出映射模板
└────────────┬───────────────┘
             │
             ▼
┌────────────────────────────┐
│  用户环节：填写映射表        │
│  - 对于待匹配项，用户在导出的模板中指定：
│    → 最终保留哪个编码，或指定映射关系
└────────────┬───────────────┘
             │
             ▼
┌────────────────────────────┐
│  第三步：应用用户映射        │
│  - 根据用户填写的映射关系合并剩余产品
└────────────┬───────────────┘
             │
             ▼
┌────────────────────────────┐
│  第四步：生成输出文件        │
└────────────┬───────────────┘
             │
             ▼
┌────────────────────────────┐
│  第五步：输出详细统计日志    │
│  - 原始总数、自动匹配数、单方独加数、待匹配数
│  - 记录所有处理过程便于排查
└────────────────────────────┘
```

## 输出文件

| 输出文件 | 格式 | 内容说明 |
|---------|------|---------|
| `product_unified.xlsx`/`.csv` | Excel + CSV | 最终统一产品库，包含：统一编码、规格、售价、来源标记 |
| `code_mapping.xlsx` | Excel | 完整映射表：原始文件、原始编码 → 统一编码 |
| `initial_stock_updated.xlsx` | Excel | 更新后的库存文件，产品编码已替换为统一编码 |
| `unify_products.log` | 日志文件 | 详细处理日志和统计数据 |

## 输出字段说明

### product_unified.xlsx
| 列名 | 说明 |
|------|------|
| unified_code | 统一后的产品编码 |
| product_name | 产品名称（如有） |
| spec | 规格 |
| unit | 单位 |
| cost_price | 成本价 |
| sale_price | 售价 |
| category | 分类 |
| source | 来源：file1_only / file2_only / both |

### code_mapping.xlsx
| 列名 | 说明 |
|------|------|
| source_file | 来源文件（file1/file2） |
| original_code | 原始编码 |
| unified_code | 统一后编码 |
| needs_mapping | 是否需要人工映射（是/否） |

## 统计信息输出

程序启动和结束时会输出以下统计：
