# 产品报价Web应用设计文档

**日期**: 2026-04-29
**版本**: v1.0
**状态**: 设计确认

---

## 一、项目概述

将现有的产品报价技能封装为Web应用，支持：
- 用户通过浏览器上传报价单
- 系统自动进行产品编码匹配和价格填充
- 返回填好价格的报价单供下载
- 用户填写无匹配产品价格后回传
- 自动更新标准产品库并反馈结果

## 二、部署环境

| 项 | 值 |
|----|----|
| 部署方式 | 公网IP访问 |
| 用户模式 | 单用户，密码保护 |
| 技术栈 | Flask + Jinja2模板 + 原生JavaScript |
| 目标目录 | 独立项目文件夹 `product_quoting_web/` |

## 三、整体架构

```
用户浏览器 → Flask Web服务 → 报价技能核心 → 文件系统
     ↑             ↓              ↓
     └────── 页面渲染/文件上传/结果下载 ───────┘
```

### 核心工作流

1. 用户访问首页 → 看到文件上传表单
2. 上传Excel/CSV报价单 → 选择产品列和数量列
3. 提交 → 后台调用 quoting skill 进行匹配处理
4. 返回结果页面 → 显示匹配率统计、可下载报价结果
5. 用户下载Excel → 人工填写无匹配产品价格
6. 用户在"更新标准库"页面上传已填写的Excel
7. 后台自动识别新增产品 → 更新标准库 → 显示更新结果

## 四、页面路由与功能

| 路由 | 页面 | 功能 |
|------|------|------|
| `/` | 首页 | 上传报价单 + 列选择 |
| `/result/<job_id>` | 报价结果页 | 显示统计、下载文件、查看未匹配项 |
| `/update` | 标准库更新页 | 上传已填写价格的文件、列选择 |
| `/update/result/<job_id>` | 更新结果页 | 显示新增/跳过数量、详情 |
| `/library` | 标准库概览 | 查看编码总数、最近更新记录 |

## 六、安全设计

### 访问控制
- HTTP Basic Auth 密码保护
- 密码存储在环境变量 `QUOTING_PASSWORD`
- 启动时显示访问URL和密码提示

### 文件安全
- 支持格式: `.xlsx`, `.xls`, `.csv`
- 文件大小限制: 50MB
- 文件名安全过滤（去除路径、特殊字符）
- 不执行文件中的宏/VBA

### 标准库保护
- 更新前自动备份到 `backup/` 目录
- 保留最近5个备份版本
- 提供回滚功能（可恢复到上一个版本）

## 七、核心交互流程

### 报价流程

1. **上传页面**
   - 拖放或点击选择文件
   - 上传后预览前几行数据（表格显示）
   - 下拉选择产品编码列和数量列
   - 点击"开始报价"按钮

2. **处理中**
   - 显示加载动画
   - 处理完成后自动跳转结果页

3. **结果页**
   - 统计卡片：总行数、匹配数、匹配率
   - 各类匹配规则的详细统计
   - "下载报价结果"按钮
   - 未匹配编码列表（可折叠）

### 更新标准库流程

1. 上传已填写价格的Excel
2. 选择：产品列、价格列、匹配标注列
3. 预览新增条目
4. 确认后执行更新
5. 结果页显示：新增XX个，跳过XX个（已存在），具体列表

## 八、代码复用策略

- **直接复用现有 quoting_skill 模块**
- Web层仅负责：文件接收、参数传递、结果展示
- 核心匹配逻辑完全复用于现有技能，不重复实现

## 九、部署说明

```bash
# 1. 进入项目目录
cd product_quoting_web

# 2. 安装依赖
pip install -r requirements.txt

# 3. 设置密码（可选，默认有随机密码）
export QUOTING_PASSWORD="your-password"

# 4. 启动服务
python app.py

# 5. 访问: http://服务器IP:5000
```
