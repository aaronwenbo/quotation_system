# 产品编码匹配规则维护指南

**版本**: 1.0
**更新日期**: 2026-04-28

## 概述

本项目的产品编码匹配规则采用**配置与代码分离**的方式维护。所有规则定义在独立的配置文件中，非技术人员也可以轻松添加和修改匹配规则。

---

## 配置文件位置

```
data/matching_rules_config.json
```

> 💡 **提示**：这是一个标准的 JSON 文件，可以用任何文本编辑器或 VS Code 打开编辑。

---

## 规则五（前五位编码匹配表）维护

### 1. 规则说明

规则五用于前五位编码的映射匹配，适用于：
- 3开头系列 → 2开头系列的转换
- 其他固定前缀的产品编码匹配

### 2. 如何添加新的匹配对

编辑 `data/matching_rules_config.json`，在 `rule5_prefix_mapping.mappings` 中添加新行：

```json
{
  "rule5_prefix_mapping": {
    "description": "规则五：前五位编码匹配表",
    "note": "格式: '原编码前五位(无-)': ['目标编码前五位(无-)', '匹配说明']",
    "mappings": {
      "30411": ["20411", "30411→20411"],
      "30511": ["20511", "30511→20511"],
      "34211": ["24211", "34211→24211"],
      "30211": ["20211", "30211→20211"],
      "30611": ["20611", "30611→20611"],
      "在这里添加": ["目标编码", "显示的匹配说明"]   // ← 新增行
    }
  }
}
```

### 3. 格式要求

| 字段 | 说明 | 示例 |
|------|------|------|
| **Key** | 原编码前五位（去掉所有 `-`） | `"30411"` |
| **Value[0]** | 目标标准编码前五位（去掉所有 `-`） | `"20411"` |
| **Value[1]** | 匹配说明（用于日志输出） | `"30411→20411"` |

### 4. 注意事项

⚠️ **重要：**
1. JSON 文件不允许最后一项后面有逗号
2. 所有引号必须是英文双引号 `"`
3. 前五位必须是**去掉 `-` 后的前五个字符**
4. 修改后建议运行一次报价验证效果

## JSON 格式验证

如果不确定 JSON 格式是否正确，可以：

**方式一：在线验证**
- 访问 https://jsonlint.com/
- 粘贴文件内容，点击 "Validate JSON"

**方式二：命令行验证**
```bash
cd src
python -c "import json; json.load(open('../data/matching_rules_config.json')); print('✅ JSON 格式正确')"
```

## 常见问题

### Q: 添加了新的匹配对，但没有生效？
A: 检查以下几点：
1. JSON 文件格式是否正确（逗号、引号）
2. 前五位编码是否正确（去掉 `-` 后取前五位）
3. 重新运行报价脚本

### Q: 可以同时配置双向匹配吗？
A: 可以。例如同时添加 `30411→20411` 和 `20411→30411` 两条配置。

### Q: 配置文件需要备份吗？
A: 建议重要修改前备份一份，或使用 git 进行版本管理。
